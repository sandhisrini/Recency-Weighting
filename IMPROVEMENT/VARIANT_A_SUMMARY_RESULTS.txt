WHAT VARIANT WAS COMPLETED:
================================================================================

VARIANT A: Recency-Weighted Scoring

This is an improvement to the TR-Rules temporal knowledge graph completion method.
It modifies how candidate entities are scored by weighting recent historical facts
more heavily than old facts.

Core Change:
  Original TR-Rules: All body instances (historical facts) count equally when 
                     scoring candidates
  
  Variant A:        Recent body instances count MORE than old ones using 
                    exponential decay weighting

The Principle:
  "Recent facts are better predictors of future events than old facts"
  
  Example:
    If USA attacked Russia in 2010 AND USA attacked Russia in 2013,
    the 2013 attack is more relevant for predicting 2014 actions


================================================================================
HOW VARIANT A WORKS - SIMPLE EXPLANATION:
================================================================================

Step 1: Extract Timestamps
   For each candidate, get timestamps of all matching historical facts
   Example: China attacked by USA in (2010, 2011, 2013)

Step 2: Calculate Recency Scores
   Convert each timestamp to a recency weight:
   recency_score = exp(-0.3 * years_old)
   
   If current year is 2014:
     2010 fact: exp(-0.3 * 4) = 0.301 (30% weight - OLD)
     2011 fact: exp(-0.3 * 3) = 0.407 (41% weight - MEDIUM)
     2013 fact: exp(-0.3 * 1) = 0.741 (74% weight - RECENT!)

Step 3: Average Recency
   avg_recency = (0.301 + 0.407 + 0.741) / 3 = 0.450

Step 4: Apply Boost
   boost_factor = 1 + avg_recency = 1 + 0.450 = 1.450
   
   This means: Boost the score by 45% because recent facts support this candidate

Step 5: Calculate Final Score
   score = base_confidence + (1 - a) * time_decay * boost_factor
   
   With default parameters a=0.5:
   score = 0.5 * rule_confidence + 0.5 * time_decay * 1.450

Result: Candidates with RECENT supporting evidence get higher scores

================================================================================
DETAILED METRIC EXPLANATION:
================================================================================

What is Hits@K?
  Percentage of correct answers that rank in top K positions
  
  Hits@1 = 33.91%: In about 1 of 3 test queries, we get the PERFECT prediction
  Hits@3 = 47.78%: In about 1 of 2 test queries, correct answer is in top 3
  Hits@10 = 61.93%: In about 2 of 3 test queries, correct answer is in top 10


What is MRR (Mean Reciprocal Rank)?
  Average of (1 / rank) across all test queries
  
  If correct answer ranks at position 1: contributes 1/1 = 1.0 (perfect)
  If correct answer ranks at position 2: contributes 1/2 = 0.5
  If correct answer ranks at position 3: contributes 1/3 = 0.33
  If correct answer ranks at position 10: contributes 1/10 = 0.1
  
  Your MRR = 0.4326 means on average you rank correct answers at position ~2.31
  This is good - much better than random (~0.06)
